Inventory Managament System

Codigo Fonte: Privado

O Armazem paraíba, assim como empresas de varejo, devido ao seu tamanho e porte, possui um sistema de Inventory Managament, que continuamente está sendo melhorado, de modo a otimizar ao máximo os custos bem conhecidos na teoria de inventário como holding cost, custo de transporte, dentre outros.

Contributions

Para esse projeto, particei de uma dupla com desenvolvedor Armando Neto que me auxiliou na criação de uma API REST capaz de enviar a um sistema maior de gerenciamento de inventário dados uma previsão de vendas ou uma sugestão de ponto de produto, seguindo essa previsão e outras regras de negócio.

PreProcessing

Essa foi uma parte bastante desafiadora dentre as minhas contribuições para o projeto, devido a alguns problemas com a base e devido ao fato de que apenas recenmente um banco de dados foi implementado nesse sistema, logo tive nessa etapa que lidar com algumas diferença de normalização entre os dados no decorrer do histórico, além de lidar com erros humanos.

Data Pipeline

Para garantir que os problemas encontrados no preprocessamento não se repetissem, também criei um pipeline de dados para garantir um melhor ambiente para futuros cientistas de dados que forem manipular essas bases.

Sales Forecast

Com o ambiente pronto, gerenciei uma série de métodos procurando um que se melhor encaixasse no problema de vendas para os dados que me eram disponíveis, dentre os usados, aqueles que obteram melhores resultados para esse problema como informado em um artigo cientifico que em breve estará disponível foram XGBoost e Arima

API REST

Com a finalização dessas etapas, otimização dos modelos eu criei um API REST para que esse sistema pudesse ser integrado a um já existente utilizando o microframework Flask para ter maior controle e segurança na hora da anexação de certas regras de negócio.
